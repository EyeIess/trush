This is the meeting agenda/notes for the Meeting held on 2009-12-14.

Date and Time: 2009-12-14 20:00 UTC

Agenda

  • Next Meeting Date
    • Monday 11th Jan
  • Releases
    • Wes notes a linux binary for makerelease now available
      • Based on Par::Packer
      • Dave will look at this and/or RPM-based packageing
    • 5.2.x -- dts12
    • 5.4.x -- dts12
      • pre2 versions should appear in the next week or so
  • Coding Issues
    • magfr's frustration with OID registrations and his hopefully-this-time solution proposal
      • Held over to the next meeting
    • TrogL's multi-core CPU monitoring
      • Attempting to ascertain the exact user requirements
      • Some progress, but rather slow
    • Dave - tracker patch to support monitoring for "unwanted" processes
      • Involves change in interpretation of min=max=0
      • OK for main trunk (?with warning) - don't backport
  • Testing infrastructure and supported releases
    • Wes' proposal from a few meetings ago
      • Past: Wes will try and put something together in Q1 2010
      • Held over to the next meeting
  • Future Directions
    • TrogL: Interested in updating UCD MIB data
      • Tackle one chunk at a time
      • netSnmp tree includes an (empty) framework for this
      • Common data handling can support multiple MIB interfaces (UCD/netSnmp/IETF-standard/etc)
  • Other Issues
    • How to handle (new) copyright notices?
      • Full-length copyright notices should go into COPYING
      • Short (3-line) notices acceptable in code files
      • Shouldn't affect doxygen output
    • Welcome to Bart
    • Review process for new patches?
      • Developers are fine to apply patches
        Use your professional judgement
      • If in doubt, raise any concerns on the coders list
        (Old hands, take note!)
  • Round Table
    • Wes: DTLS
    • magfr: some merging to do. agentxtrap
    • TrogL: new documentation and trap handler (promised for January)
    • Dave: 5.[24].x and bug/patch tracker bashing
    • Bart: Call for feedback re: recent patches
      (esp. cached noSuchInstance support)
